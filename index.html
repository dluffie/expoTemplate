Project Title
AI-Assisted College Activity Point Management System
📖 Introduction

Colleges require students to earn activity points through internships, workshops, sports, and technical events. Currently this process is mostly manual, causing delays, document mismanagement, and verification difficulties.

The proposed system is an AI-Assisted Activity Point Management System that allows students to upload certificates, automatically extract activity details using AI, and submit them for teacher verification — providing a structured digital workflow for students, teachers, and administrators.

🎯 Objectives
  • Develop a centralized system for managing student activity points
  • Allow students to upload certificates for verification
  • Use AI (Google Gemini) to extract activity details from documents automatically
  • Enable teachers to verify and approve or reject activity submissions
  • Provide dashboards for tracking total activity points per student
  • Send email reminders for pending submissions and approvals
  • Implement role-based access control for Student, Teacher, and Admin
  • Support faculty-initiated activity submissions on behalf of students
  • Ensure data integrity with snapshot-based activity records
👥 System Users

1. Students — Upload certificates, track earned activity points, update profile, and receive submission status notifications.

2. Teachers / Class Tutors — Verify student activity submissions, approve or reject with remarks, and submit activities on behalf of students.

3. Administrator — Manage users, assign teachers to classes, configure rules engine, and monitor system-wide activity.

🛠️ Technologies Used
  • Frontend: React.js, Tailwind CSS
  • Backend: Node.js + Express.js
  • Database: MongoDB (Mongoose)
  • AI / OCR: Google Gemini API + OCR fallback
  • File Storage: AWS S3 (Presigned Uploads)
  • Cache / Queue: Redis + Bull Queues
  • Auth: JWT + Email OTP Verification
⚡ Key Features
  • Student registration with email OTP verification
  • Activity certificate upload with AI-powered data extraction (Gemini API)
  • OCR fallback for scanned / low-quality documents
  • Teacher verification panel — approve, reject, or request resubmission
  • Faculty submission for student activities
  • Email reminders for pending submissions and approvals
  • Activity points calculation with configurable rules engine
  • Tiered point modifiers based on activity category and level
  • Role-based access control (Student / Teacher / Admin)
  • Timetable CSV sync for teacher class management
  • Admin dashboard — monitor all submissions, users, and stats
  • Permanent activity snapshots for data integrity

✅ Expected Outcome

The system will streamline activity point management by digitizing certificate submission, automating data extraction using AI, and enabling efficient verification workflows — reducing administrative workload while giving students a transparent way to track their academic progress.

🏗️ Final Structure

Client Layer

  • React.js Web App — Student / Teacher / Admin portals

API Layer

  • Node.js + Express REST API with JWT authentication
  • Role-based middleware for route protection

AI / Processing Layer

  • Google Gemini API → extract activity name, date, category from certificate
  • OCR fallback for scanned documents
  • Bull Queue for background processing jobs

Data Layer

  • MongoDB — Users, Activities, Submissions, Snapshots
  • Redis — Caching, session tokens, queue management
  • AWS S3 — Secure certificate file storage
⚠️ Important Subproblems
  • Handling low-quality or handwritten certificate scans accurately
  • Preventing duplicate activity submissions
  • Designing a flexible rules engine for variable point categories
  • Ensuring secure presigned file uploads to AWS S3
  • Managing role-based access without breaking teacher workflows
